## 📄 ОСНОВНЫЕ ФАЙЛЫ

### 1. **database.py** (280 строк кода)
**Назначение:** Управление базой данных SQLite и всеми CRUD операциями

**Основной класс:**
- `DatabaseManager` - все операции с БД

**Основные методы:**
```python
__init__(db_file: str)           # Инициализация и создание таблицы
add_request(...)                 # Добавить заявку
get_all_requests() -> List       # Получить все заявки
get_request_by_id(id) -> Tuple   # Получить заявку по ID
update_request(...) -> bool      # Обновить заявку
delete_request(id) -> bool       # Удалить заявку
search_by_client_name(name)      # Поиск по имени клиента
filter_by_status(status)         # Фильтрация по статусу
update_status(id, status) -> bool # Изменить статус
get_status_counts() -> Dict      # Получить статистику
```

**Особенности:**
- Полная обработка ошибок (try/except)
- Валидация обязательных полей
- Type hints для всех параметров
- Подробные docstrings
- Работает с SQLite3 (встроено в Python)

**Использование:**
```python
from database import DatabaseManager

db = DatabaseManager("design_requests.db")
db.add_request("Client Name", "contact@email.com", "Logo Design", "Desc", "2026-03-01")
```

### 2. **main.py** (530 строк кода)
**Назначение:** GUI приложения на Tkinter и точка входа

**Основной класс:**
- `DesignRequestApp` - главное окно приложения

**Основные методы:**
```python
setup_gui()              # Инициализация интерфейса
add_request()            # Добавить заявку
update_request()         # Обновить заявку
delete_request()         # Удалить заявку
search_requests()        # Поиск по имени
filter_requests()        # Фильтрация по статусу
change_status()          # Изменить статус
load_requests()          # Загрузить заявки в таблицу
validate_required_fields() -> bool   # Валидация
validate_deadline_format(date) -> bool # Валидация даты
clear_fields()           # Очистить поля
```

**Интерфейс:**
- Ввод данных (поля и выпадающие списки)
- Таблица Treeview со всеми заявками
- Кнопки управления (Add, Update, Delete, etc.)
- Поиск и фильтрация
- Цветовое кодирование статусов

**Используемые компоненты Tkinter:**
- Frame, LabelFrame - контейнеры
- Entry - однострочные поля ввода
- Combobox - выпадающие списки
- ScrolledText - многострочное описание
- Treeview - таблица данных
- Button - кнопки
- Label - подписи
- messagebox - диалоги

**Использование:**
```python
if __name__ == "__main__":
    root = tk.Tk()
    app = DesignRequestApp(root)
    root.mainloop()
```

### **design_requests.db**
SQLite база данных, создается автоматически при первом запуске.

**Структура таблицы:**
```sql
CREATE TABLE design_requests (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    client_name TEXT NOT NULL,
    contact_info TEXT,
    project_type TEXT NOT NULL,
    description TEXT,
    status TEXT DEFAULT 'New',
    deadline TEXT,
    created_at TEXT NOT NULL
)
```
