# DesignRequest - Project Summary

## Описание проекта

Полнофункциональное настольное приложение для управления заявками на дизайн-проекты, разработанное на Python с использованием Tkinter и SQLite.

## Реализованные требования

### ✅ Основные функции

1. **SQLite База данных**
   - Файл базы данных: `design_requests.db`
   - Таблица `design_requests` с полями:
     - id (INTEGER PRIMARY KEY AUTOINCREMENT)
     - client_name (TEXT)
     - contact_info (TEXT)
     - project_type (TEXT)
     - description (TEXT)
     - status (TEXT)
     - deadline (TEXT)
     - created_at (TEXT)

2. **CRUD операции**
   - ✅ Add (добавление новой заявки)
   - ✅ Read (просмотр заявок в таблице)
   - ✅ Update (редактирование заявки)
   - ✅ Delete (удаление заявки)

3. **Функциональность**
   - ✅ Добавление новой заявки
   - ✅ Редактирование выбранной заявки
   - ✅ Удаление заявки с подтверждением
   - ✅ Изменение статуса заявки
   - ✅ Поиск по имени клиента
   - ✅ Фильтрация по статусу
   - ✅ Отображение всех заявок в Treeview таблице
   - ✅ Валидация обязательных полей
   - ✅ Валидация формата даты (YYYY-MM-DD)

4. **Статусы заявок**
   - New (новая)
   - In Progress (в работе)
   - On Review (на рецензии)
   - Completed (завершена)

### ✅ Структура программы

1. **database.py** - модуль базы данных
   - Класс DatabaseManager с методами CRUD
   - Функции поиска и фильтрации
   - Обработка ошибок try/except
   - Валидация данных

2. **main.py** - GUI класс и точка входа
   - Класс DesignRequestApp для Tkinter интерфейса
   - Методы для всех функций: add, edit, delete, search, filter
   - Валидация входных данных
   - Обработка ошибок
   - Функция main() и точка входа

3. **config.py** - конфигурационные параметры
   - Константы приложения
   - Стили и размеры
   - Сообщения и тексты ошибок

### ✅ Интерфейс

1. **Панель ввода данных**
   - Поле "Client Name" (обязательное)
   - Поле "Contact Info"
   - Выпадающий список "Project Type" (обязательное)
   - Выпадающий список "Status"
   - Поле "Deadline"
   - Текстовое поле "Description" (многострочное)

2. **Панель поиска и фильтрации**
   - Поле поиска по имени клиента
   - Кнопка "Search"
   - Выпадающий список фильтрации по статусу
   - Кнопка "Show All"

3. **Таблица (Treeview)**
   - 7 столбцов: ID, Client Name, Contact, Project Type, Status, Deadline, Created At
   - Цветовое кодирование по статусу
   - Выделение выбранной строки
   - Автоматическое заполнение полей при выборе

4. **Кнопки управления**
   - Add Request - добавление новой заявки
   - Update Request - обновление выбранной заявки
   - Delete Request - удаление выбранной заявки
   - Change Status - изменение статуса
   - Clear Fields - очистка полей
   - Show All - отображение всех заявок
   - Exit - выход из приложения

- delete_request() - удаление заявки

- filter_by_status() - фильтрация по статусу
- update_status() - обновление статуса
- get_status_counts() - получение статистики

Основной класс GUI приложения:
- setup_gui() - инициализация интерфейса

- update_request() - метод обновления
- delete_request() - метод удаления
