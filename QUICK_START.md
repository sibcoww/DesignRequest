## Первые шаги в приложении

### 1️⃣ Добавьте первую заявку (2 мин)

```
1. В окне приложения найдите "Request Details" раздел
2. Заполните два ОБЯЗАТЕЛЬНЫХ поля:
   • Client Name: "Мой первый клиент"
   • Project Type: выберите "Logo Design" из выпадающего списка
3. Опционально добавьте:
   • Contact Info: "client@email.com"
   • Deadline: "2026-03-30" (формат: YYYY-MM-DD)
   • Description: "Описание проекта..."
4. Нажмите кнопку "Add Request"
5. Вы должны увидеть зеленое сообщение "Request added successfully!"
```

### 2️⃣ Просмотрите заявку в таблице

```
1. Прокрутите окно вниз до таблицы
2. Вы должны увидеть вашу заявку в первой строке
3. Заметьте цветное выделение (красное = новая заявка)
```

### 3️⃣ Отредактируйте заявку

```
1. Кликните на вашу заявку в таблице
2. Поля автоматически заполнятся данными заявки
3. Измените что-нибудь (например, статус)
4. Нажмите "Update Request"
5. Таблица обновится с новыми данными
```

### 4️⃣ Измените статус

```
1. Выберите заявку в таблице
2. В разделе "Request Details" измените "Status" на "In Progress"
3. Нажмите кнопку "Change Status"
4. Цвет строки в таблице изменится с красного на желтый
```

### 5️⃣ Попробуйте поиск

```
1. В поле "Search by Client" введите часть имени клиента
2. Нажмите "Search"
3. Таблица отфильтруется только для этого клиента
4. Нажмите "Show All" чтобы вернуться к полному списку
```

## Быстрая справка по кнопкам

| Кнопка | Что делает | Клавиша |
|--------|-----------|---------|
| **Add Request** | Добавляет новую заявку | - |
| **Update Request** | Сохраняет изменения выбранной заявки | - |
| **Delete Request** | Удаляет выбранную заявку | - |
| **Change Status** | Меняет статус выбранной заявки | - |
| **Clear Fields** | Очищает все поля ввода | - |
| **Search** | Ищет по имени клиента | - |
| **Show All** | Показывает все заявки | - |
| **Exit** | Закрывает приложение | Alt+F4 |

## Частые вопросы

### Q: Где хранятся мои данные?
**A:** В файле `design_requests.db` в папке приложения. Это локальный файл на вашем компьютере.

### Q: Как сделать резервную копию?
**A:** Просто скопируйте файл `design_requests.db` в безопасное место.

### Q: Можно ли восстановить удаленную заявку?
**A:** Нет, удаление постоянное. Поэтому всегда проверяйте перед удалением!

### Q: Сколько заявок может хранить приложение?
**A:** Практически неограниченно (тысячи).

### Q: Работает ли без интернета?
**A:** Да! Приложение работает полностью офлайн.

### Q: Можно ли использовать на другом компьютере?
**A:** Да, просто скопируйте всю папку DesignRequest и запустите там.

## Если что-то пошло не так

### Ошибка: "Python not found"
```
✅ Решение: Установите Python с python.org
```

### Ошибка: "No module named tkinter"
```
Windows: Переустановите Python, выбрав опцию "tcl/tk"
macOS: Обычно включено, переустановите Python
Linux: sudo apt-get install python3-tk (Ubuntu/Debian)
```

### Приложение не запускается
```
1. Убедитесь что вы в правильной папке
2. Проверьте что оба файла на месте: database.py и main.py
3. Попробуйте: python main.py (вместо двойного клика на run.bat)
```

### Сообщение об ошибке при добавлении заявки
```
1. Убедитесь что заполнены оба обязательных поля
2. Проверьте формат даты (YYYY-MM-DD)
3. Попробуйте закрыть и переоткрыть приложение
```
