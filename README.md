# DesignRequest - Project Request Management System

Полнофункциональное настольное приложение для управления заявками на дизайн-проекты, разработанное на Python с использованием Tkinter и SQLite.

## Требования

- Python 3.7+
- Tkinter (обычно поставляется с Python)
- SQLite3 (встроено в Python)

## Установка

1. Убедитесь, что Python установлен на вашем компьютере.
2. Tkinter обычно включен в стандартную установку Python.

## Запуск приложения

Откройте терминал/командную строку в директории проекта и выполните:

```bash
python main.py
```

## Структура проекта

### Файлы

- **database.py** - Модуль для работы с базой данных
  - Класс `DatabaseManager` управляет всеми операциями SQLite
  - Реализованы все CRUD операции (Create, Read, Update, Delete)
  - Функции поиска и фильтрации

- **main.py** - Основной модуль приложения
  - Класс `DesignRequestApp` реализует GUI на Tkinter
  - Все функции пользовательского интерфейса
  - Валидация входных данных
  - Точка входа в приложение

- **design_requests.db** - Локальная база данных SQLite (создается автоматически)

## Функциональность

### Основные возможности

1. **Добавление новой заявки**
   - Обязательные поля: Имя клиента, Тип проекта
   - Опциональные поля: Контактная информация, Описание, Дедлайн
   - Статус по умолчанию: "New"

2. **Редактирование заявки**
   - Выберите заявку из таблицы
   - Отредактируйте необходимые поля
   - Нажмите "Update Request"

3. **Удаление заявки**
   - Выберите заявку из таблицы
   - Нажмите "Delete Request"
   - Подтвердите удаление

4. **Изменение статуса**
   - Выберите заявку и новый статус
   - Нажмите "Change Status"
   - Доступные статусы: New, In Progress, On Review, Completed

5. **Поиск по имени клиента**
   - Введите часть имени клиента
   - Нажмите "Search"
   - Результаты будут отфильтрованы в таблице

6. **Фильтрация по статусу**
   - Выберите статус из выпадающего меню
   - Таблица автоматически отфильруется
   - Выберите "All" для отображения всех заявок

7. **Просмотр всех заявок**
   - Нажмите "Show All" для отображения всех заявок

### Поля базы данных

```
id (INTEGER PRIMARY KEY AUTOINCREMENT) - Уникальный идентификатор
client_name (TEXT) - Имя клиента (обязательное)
contact_info (TEXT) - Контактная информация
project_type (TEXT) - Тип проекта (обязательное)
description (TEXT) - Описание проекта
status (TEXT) - Статус заявки (New, In Progress, On Review, Completed)
deadline (TEXT) - Дедлайн проекта (формат: YYYY-MM-DD)
created_at (TEXT) - Дата и время создания
```

## Типы проектов

По умолчанию доступны следующие типы проектов:
- Logo Design (Дизайн логотипа)
- Web Design (Веб-дизайн)
- Mobile App (Мобильное приложение)

## Статусы заявок

- **New** - Новая заявка (подсвечивается красным)
- **In Progress** - В работе (подсвечивается желтым)
- **On Review** - На рецензии (подсвечивается голубым)
- **Completed** - Завершена (подсвечивается зеленым)

## Особенности

- **Валидация данных**: Проверка обязательных полей и формата даты
- **Обработка ошибок**: Использование try/except блоков
- **Цветовое кодирование**: Строки таблицы окрашены в зависимости от статуса
- **Удобный интерфейс**: Интуитивный дизайн с понятными кнопками
- **Локальная БД**: Все данные хранятся локально в SQLite

## Решение проблем

### Ошибка "No module named 'tkinter'"

На Linux системах нужно установить Tkinter отдельно:
```bash
# Для Ubuntu/Debian
sudo apt-get install python3-tk

# Для Fedora
sudo dnf install python3-tkinter

# Для Arch Linux
sudo pacman -S tk
```

### Ошибка при запуске

Убедитесь, что:
1. Вы находитесь в правильной директории
2. Python 3.7+ установлен
3. Файлы `database.py` и `main.py` находятся в одной директории
4. У вас есть права на запись в директории для создания БД
